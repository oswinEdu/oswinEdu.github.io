    异步 #

    1.Unity异步 #

    缺点

    1. 没有返回值
    2. 依赖 GameObject

    特点

    1. Unity本身是单线程运行的, gameObject不是激活状态, 对应的副线程不会执行
    2. Unity主线程里的副线程, 主线程执行完渲染, 副线程执行, Coroutine的本质是Unity在主线程上模拟了多线程
    3. 需要在迭代器中加入yield return让副线程能够跳出

    2.C#异步 #

    缺点

    1. 导出 Webgl 不能使用

    特点

    1. Task是C#自带的多线程方法
    2. 可以使用async void xx（）和async Task xx（）两种实现方法
    3. Task所创建的进程独立于Unity，也就是说Unity停止运行后Task不一定会立刻停止，因此可能造成找不到GameObject的报错
    4. 需要使用awit 来跳出该进程（类似于协程的yield retrun）
    5. 也可以使用await Task.Delay（int，单位为毫秒）来延迟运行，实现计时器的效果。

    3.UniTask #

    4.Thread #

    特点

    1. Thread是真正的多线程，与Task、Coroutine有本质区别, 堵塞 Thread 线程不影响主线程
    2. Thread内只能进行传值，不能传递Unity内的Object
    3. 可以在Thread内修改的：int、float、string、bool、struct、Vector3、List、Dictionary等
    4. 使用Thread.Sleep(int，单位为毫秒)来延迟运行，实现计时器的效果。
